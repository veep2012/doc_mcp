# Technical Debt

Deferred implementation gaps and design compromises captured during development.

## TD-20260607-01
- Title: Decouple tool search mode from vectorizer configuration
- Context: While working on search behavior in `src/docmcp/tools.py` and config normalization in `src/docmcp/config/loader.py`, the current `search_engine` setting still controls search routing too tightly for the MCP tools.
- Location: `src/docmcp/tools.py`, `src/docmcp/config/loader.py`, `documentation/configuration.md`, `documentation/mcp-server.md`
- Impact/Risk: Callers cannot independently choose keyword, vector, or hybrid search at query time when the site-level configuration is set differently. This reduces flexibility and can force a suboptimal retrieval mode for specific requests.
- Why Deferred Now: Fixing this properly requires a query-time API and contract change, plus corresponding test and documentation updates, which is outside the current scope.
- Proposed Fix: Add an explicit search-mode override or resolver for MCP tools so the agent can choose keyword, vector, or hybrid search independently of the vectorizer-facing `search_engine` parameter.
- Acceptance Signal: The search tool accepts and respects per-request mode selection, tests verify that query-time mode is independent from vectorizer configuration, and the docs describe the override behavior clearly.
- Status: open
