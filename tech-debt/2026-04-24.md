## TD-20260424-01
- Title: Support reindexing only a selected list of pages
- Context: The current reindex flow is site-scoped and crawls from the configured start URL, which forces a full-site pass even when only a few pages need to be refreshed.
- Location: `crawl_cli.py`, `src/docmcp/tools.py`, `document/requirements.md`
- Impact/Risk: Reindexing large sites is slower than necessary, increases crawl load, and makes targeted refreshes impossible for users who already know the exact pages that changed.
- Why Deferred Now: The immediate task is to record the limitation, not redesign the crawler/API surface or add the supporting tests and storage behavior.
- Proposed Fix: Add a page-selection parameter to the reindex command/tool so callers can supply a list of page URLs or canonical page IDs; update the crawl flow to visit only those pages and keep the index behavior consistent for the selected subset.
- Acceptance Signal: `reindex_site` or its CLI equivalent accepts an explicit page list and reindexes only those pages, with tests proving non-selected pages are left untouched.
- Status: open
