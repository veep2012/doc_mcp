# Technical Debt Log

Deferred items discovered while implementing the MCP version tool and verifying the test suite.

## TD-20260509-01
- **Title:** Make smoke test collection tolerant of optional runtime dependencies
- **Context:** While validating the new `get_version` tool, full `pytest` collection failed before execution because smoke and CLI tests depend on packages not present in this environment, including `mcp` and `playwright`.
- **Location:** `tests/smoke/support.py`, `tests/smoke/test_crawl_smoke.py`, `tests/smoke/test_mcp_smoke.py`, `tests/test_crawl_cli.py`
- **Impact/Risk:** The repository cannot be collected cleanly in lightweight environments, so contributors may see import-time failures before they reach the tests relevant to their change.
- **Why Deferred Now:** The version tool change does not depend on the smoke suite, and fixing the broader dependency story would expand scope beyond the current task.
- **Proposed Fix:** Gate smoke and CLI tests behind optional dependency checks or explicit skip markers, and make the test suite import-safe when Playwright or MCP are unavailable.
- **Acceptance Signal:** `pytest -q` skips the smoke/CLI tests cleanly when optional runtime dependencies are missing and completes collection without import errors.
- **Status:** open

## TD-20260509-02
- **Title:** Remove direct `tests.*` imports that break collection outside pytest package mode
- **Context:** Full test collection also failed on imports like `from tests.conftest import REPO_ROOT` in smoke tests.
- **Location:** `tests/smoke/support.py`, `tests/smoke/test_crawl_smoke.py`, `tests/smoke/test_mcp_smoke.py`, `tests/test_smoke_support.py`
- **Impact/Risk:** Collection depends on `tests` being importable as a package, which is fragile and can fail in standard local or CI runs.
- **Why Deferred Now:** The failure is unrelated to the MCP version tool and would require a broader test-layout cleanup.
- **Proposed Fix:** Switch those imports to relative imports within the test tree or introduce a supported package/testpath setup so `tests` is always importable.
- **Acceptance Signal:** The full test suite collects without `ModuleNotFoundError: No module named 'tests'`.
- **Status:** open
