# Technical Debt Log

Deferred items discovered while refining site crawling and index behavior.

## TD-20260510-01
- **Title:** Add a special config key for light, new-pages-only site indexation
- **Context:** While reviewing the crawl and index flow, it became clear that site indexing still behaves like a full crawl/update path, and there is no explicit configuration key for a lightweight mode that only indexes newly discovered pages.
- **Location:** `src/docmcp/crawl_cli.py`, `src/docmcp/index_store.py`, `config/sites.yaml.example`, `documentation/crawling.md`
- **Impact/Risk:** Sites with large or frequently updated indexes must repeat more work than necessary, which increases crawl time and makes incremental refreshes harder to control.
- **Why Deferred Now:** Defining a safe incremental indexing mode needs a small design pass around crawl state, page discovery, and how the runtime should interpret the new key.
- **Proposed Fix:** Introduce a dedicated site-level key for incremental indexing, then update the crawler to skip full reprocessing and only upsert newly discovered pages when that key is enabled.
- **Acceptance Signal:** A site can opt into a lightweight indexing mode through configuration, and a crawl run only processes new pages while leaving existing indexed pages untouched.
- **Status:** open
