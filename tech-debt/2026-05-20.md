- Impact/Risk: The current environment fails security audit because `mcp` resolves to a vulnerable `PyJWT` release. This keeps CI/security checks red and leaves the project dependent on upstream remediation.
- Why Deferred Now: The GitHub advisory referenced by `pip-audit` does not list patched versions, and PyPI currently exposes `PyJWT 2.12.1` as the latest release. There is no obvious safe version bump to apply locally without an upstream fix.
- Proposed Fix: Track upstream `PyJWT` and `mcp` releases for a patched dependency set; once a fixed release exists, update the dependency pin and re-run `make audit`.
